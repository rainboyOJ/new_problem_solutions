## 1. 题目背景与核心问题

题目描述：

FJ 想要带朋友参观农场。农场有 $N$ 块地（编号 $1 \dots N$）和 $M$ 条双向路径。

FJ 需要规划一条路线：

1. 从房子（节点 $1$）出发。
2. 到达大谷仓（节点 $N$）。
3. 最后返回房子（节点 $1$）。
4. **限制条件**：同一条路径不能走两次。
5. **目标**：让总路程最短。

核心转化：

虽然题目描述是“去再回”（$1 \to N \to 1$），但在无向图中，这等价于寻找两条从 $1$ 到 $N$ 的路径，使得：

1. 两条路径没有重复的边（Edge Disjoint）。
2. 两条路径的总长度之和最小。

## 2. 建模思路：为什么使用最小费用最大流？

这个问题可以完美地映射到**最小费用最大流 (Min-Cost Max-Flow, MCMF)** 模型上。

### 映射关系表

| **原问题要素**   | **网络流模型要素**        | **设定理由**                                                 |
| ---------------- | ------------------------- | ------------------------------------------------------------ |
| **寻找两条路径** | **最大流量限制为 2**      | 我们需要传输 2 个单位的“流”从起点到终点。                    |
| **路径不能重复** | **边容量 (Capacity) = 1** | 如果流量流过这条边，容量归零，下一条流就不能再走了。         |
| **总路程最短**   | **最小费用 (Min Cost)**   | 将路径长度设为单位流量的费用，MCMF 算法会自动寻找费用最低的流。 |
| **无向图**       | **双向建边**              | 题目中的 $u-v$ 既可以 $u \to v$ 也可以 $v \to u$。           |

## 3. 详细建图步骤 (代码逻辑解析)

结合你的代码 `farm_tour.cpp` 中的 `init()` 函数，我们来看看具体的建图过程。

### 3.1 引入超级源点和超级汇点 (关键修正)

**问题**：直接跑 $1 \to N$ 的最大流可能会导致流量超过 2（例如有 3 条路，算法可能会全部选上，导致费用偏大，不符合题目“只走两条”的要求）。

解决方案：

我们引入虚拟节点：超级源点 $S=0$ 和 超级汇点 $T=N+1$。

- **超级源点** $S \to 1$：
  - `mcmf.addEdge(0, 1, 2, 0);`
  - **容量 2**：这是系统的“总阀门”，强行限制了从 $1$ 号点出发的流量最多只有 2。
  - **费用 0**：虚拟边，不增加路程。
- **节点** $N \to$ **超级汇点** $T$：
  - `mcmf.addEdge(n, n+1, 2, 0);`
  - **容量 2**：接收 2 个单位的流量。
  - **费用 0**。

### 3.2 实际路径的建模

对于输入的一条边 `u v w`（连接 $u$ 和 $v$，长度 $w$）：

由于是无向图，我们允许流量从 $u$ 流向 $v$，或者从 $v$ 流向 $u$，但**物理上这条路只能用一次**。

在代码中，你建立了：

1. $u \to v$：`mcmf.addEdge(u, v, 1, w);`
2. $v \to u$：`mcmf.addEdge(v, u, 1, w);`

**注意**：这里的容量 1 限制了单一方向的使用。虽然我们在网络中加了两条有向边，但最大流算法只会选择其中最优的一条方向通过。如果 $u \to v$ 被使用了，反向边 $v \to u$ 在该路径中通常不会被同时使用（除非发生复杂的退流情况，那由 MCMF 内部的反向边机制处理）。

## 4. 算法运行演示

假设我们调用 `mcmf.solve(s, t)`，算法内部发生了什么？

### 第一阶段：寻找去程 (Flow = 0 -> 1)

1. **SPFA** 在残量网络中寻找从 $S$ 到 $T$ 的**最短路径**（费用最小）。
2. 假设找到了 $S \to 1 \to A \to B \to N \to T$。
3. **增广**：这条路径上所有边的容量减 1。
   - $1 \to A$ 的容量变为 0（被占用了）。
   - 总费用增加了 $Length(1 \to A \to B \to N)$。

### 第二阶段：寻找回程 (Flow = 1 -> 2)

1. **SPFA** 再次寻找最短路径。
2. 由于 $1 \to A$ 容量为 0，算法被迫寻找其他路，例如 $1 \to C \to D \to N$。
3. **反悔机制 (Regret)**：
   - 如果为了全局最优，需要“退掉”第一阶段走的某条边，算法会利用 MCMF 内部生成的**负权反向边**（例如 $B \to A$，费用 $-w$）来“退流”。
   - 这就是 MCMF 强大的地方：它不仅仅是简单的两次贪心，而是全局最优。

### 第三阶段：结束

1. 总流量达到 2。
2. $S \to 1$ 的容量耗尽，无法再找到增广路。
3. 算法结束，`min_cost` 即为答案。

## 5. 代码核心片段回顾

```cpp
// 限制总流量为 2 的关键代码
s = 0;
t = n + 1;
mcmf.init(t); 

// 超级源点 -> 1
mcmf.addEdge(s, 1, 2, 0);

// N -> 超级汇点
mcmf.addEdge(n, t, 2, 0);

// 真实路径
for (int i = 0; i < m; i++) {
    // ... 输入 u, v, w ...
    mcmf.addEdge(u, v, 1, w); // u -> v 可走一次
    mcmf.addEdge(v, u, 1, w); // v -> u 可走一次
}
```

## 6. 总结

1. **模型识别**：看到“路径不重复”、“最小总权值”、“流量固定（两条路）”，立刻想到 **最小费用最大流**。
2. **无向图处理**：拆成两条有向边，容量为 1。
3. **流量控制**：通过**超级源点**的边容量来精确控制我们需要找几条路径。如果不加超级源点，算法可能会算出流量为 3 或 4 的情况，导致 WA。
